# Roast by Jaden Order Flowcharts

## 1. 买家下单流程（烧腊部位选择）

```mermaid
flowchart TD
    A[买家进入网站] --> B[浏览烧腊菜单]
    B --> C{选择烧腊类型}

    C --> D[鸡肉类]
    C --> E[叉烧类]
    C --> F[烧鸭饭]
    C --> G[烧肉类]

    D --> D1{选择鸡肉部位}
    D1 --> D2[鸡胸]
    D1 --> D3[鸡二度]
    D1 --> D4[鸡翅]
    D1 --> D5[鸡腿]

    E --> E1{选择叉烧肥瘦}
    E1 --> E2[肥]
    E1 --> E3[瘦]
    E1 --> E4[半肥瘦]

    F --> F1[固定烧鸭饭]

    G --> G1{选择烧肉部位}
    G1 --> G2[脆皮]
    G1 --> G3[五花]
    G1 --> G4[瘦肉多]

    D2 --> H{系统检查部位状态}
    D3 --> H
    D4 --> H
    D5 --> H
    E2 --> H
    E3 --> H
    E4 --> H
    F1 --> H
    G2 --> H
    G3 --> H
    G4 --> H

    H -->|可售| I[加入购物车]
    H -->|已售完| J[显示 Sold Out / 不能选择]

    I --> K[填写姓名、电话、取餐方式、备注]
    K --> L[确认订单]
    L --> M[完成付款或提交订单]
    M --> N[订单发送到商家后台和 WhatsApp]
    N --> O[买家查看订单进度]
```

## 2. 商家管理流程（部位库存与售完控制）

```mermaid
flowchart TD
    A[商家进入后台] --> B[查看今日烧腊部位库存]
    B --> C{选择要管理的烧腊类型}

    C --> D[鸡肉部位]
    C --> E[叉烧肥瘦]
    C --> F[烧鸭饭]
    C --> G[烧肉部位]

    D --> D1[鸡胸 / 鸡二度 / 鸡翅 / 鸡腿]
    E --> E1[肥 / 瘦 / 半肥瘦]
    F --> F1[固定烧鸭饭]
    G --> G1[脆皮 / 五花 / 瘦肉多]

    D1 --> H{更新状态}
    E1 --> H
    F1 --> H
    G1 --> H

    H -->|还有库存| I[设为可售]
    H -->|卖完| J[设为已售完]
    H -->|补货或调整| K[修改库存数量]

    I --> L[网站同步显示可选择]
    J --> M[网站同步显示 Sold Out]
    K --> L

    L --> N[买家可以继续下单]
    M --> O[买家不能选择该部位]

    P[新订单进入后台] --> Q[商家确认订单]
    Q --> R[扣减对应部位库存]
    R --> S{库存是否为 0}
    S -->|是| J
    S -->|否| L
```

## 后台建议控制项

- 每个部位都可以设置为 `可售` 或 `已售完`
- 可选库存数量，例如鸡腿剩 5 份、叉烧半肥瘦剩 2 份
- 顾客端只显示可售部位，或把售完部位显示成 `Sold Out`
- 订单确认后，系统自动扣减对应部位库存
